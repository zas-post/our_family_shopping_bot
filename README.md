****Family Shopping List Telegram Bot 🛒****

Асинхронный Telegram-бот для ведения совместного семейного списка покупок в группах. Бот автоматически закрепляет актуальный список вверху чата и «тихо» обновляет его при добавлении или удалении товаров, не перегружая семейный чат лишними уведомлениями.

***✨ Основные фичи***

*Интеллектуальный парсинг:* Автоматически распознает количество и единицы измерения товара (шт, кг, л, уп и т. д.) в конце строки с помощью регулярных выражений.

*Автозакрепление и тихие обновления:* Бот держит список покупок в закрепе группы. Добавление новых позиций через префикс п+ бесшумно обновляет инлайн-кнопки в закрепленном сообщении, а само сообщение пользователя удаляется для поддержания чистоты чата.

*Интерактивный интерфейс:* Вычеркивание продуктов в один клик по инлайн-кнопке.

*Оповещение «Я в магазине!»:* Специальная кнопка, которая отправляет громкий алерт всей семье о том, что список скоро «заморозится». Защищена встроенным анти-спам механизмом (cooldown 15 минут).

*Коммерческая архитектура:* Проект полностью переведен на модульную структуру с использованием Router (в aiogram), асинхронным слоем работы с данными (Repository Pattern) и безопасным управлением конфигурацией через config.py и .env.

***🛠 Стек технологий***

Python 3.12+ (с поддержкой асинхронности)

Aiogram 3.x (Асинхронный фреймворк для Telegram Bot API)

SQLAlchemy 2.0 (Асинхронная ORM для работы с базой данных)

SQLite / Aiosqlite (Легковесная асинхронная база данных)

Alembic (Задел под миграции базы данных при масштабировании проекта)

Python-dotenv (Управление секретами и конфигурацией)

***🚀 Быстрый запуск***

1. Клонирование и подготовка окружения

Создайте виртуальное окружение и активируйте его:
```Bash
python -m venv .venv
source .venv/bin/activate  # Для Linux/macOS
# или
.venv\Scripts\activate     # Для Windows
```

2. Установка зависимостей

Установите все необходимые пакеты одной командой:
```Bash
pip install -r requirements.txt
```

3. Настройка переменных окружения

Скопируйте шаблон конфигурации и создайте личный файл .env:
```Bash
cp .env.example .env
```

Откройте созданный файл .env и вставьте ваш токен бота, полученный от @BotFather:
```Bash
BOT_TOKEN=ваша_строка_токена_сюда
```

4. Запуск бота

База данных SQLite создается и настраивается автоматически при первом запуске бота. Дополнительно настраивать таблицы не требуется!
```Bash
python main.py
```

Примечание для разработчиков: папка alembic/ настроена в проекте для будущих миграций. Если вы вносите изменения в модели SQLAlchemy (db/models.py), используйте команды alembic revision --autogenerate и alembic upgrade head для применения изменений "на лету".

***🐳 Запуск через Docker Compose (Рекомендуемый для сервера)***

Если вы разворачиваете бота на сервере (например, в LXC-контейнере на Proxmox), используйте Docker Compose для быстрой сборки и удобного логирования.

Убедитесь, что файлы ```Dockerfile```, ```.dockerignore``` и ```docker-compose.yml``` находятся в корне проекта.

Настройте файл .env, добавив туда ваш BOT_TOKEN.

Запустите сборку и старт контейнеров в фоновом режиме:
```Bash
docker compose up -d --build
```

Для просмотра логов в реальном времени через консоль сервера:
```Bash
docker compose logs -f --tail=50 family_shopping_bot
```

***📊 Мониторинг логов через веб-интерфейс Dozzle***

Для максимально удобного контроля работы бота в реальном времени развернут сервис Dozzle, предоставляющий красивый интерактивный веб-интерфейс для чтения логов прямо в браузере.

Адрес панели Dozzle: ```http://<IP_адрес_вашего_сервера>:8080``` (или иной порт, перенаправленный в вашем docker-compose.yml).

Как пользоваться: Перейдите по адресу в любом браузере, выберите контейнер family_shopping_bot в списке слева. Вы сможете наблюдать за живым логом добавления/вычеркивания продуктов, ошибками работы API и автозапуска без необходимости заходить на сервер по SSH.

***⚙️ Настройка прав бота в Telegram-группе***

Для того чтобы фичи автозакрепа и очистки чата работали корректно, добавьте бота в вашу семейную группу и наделите его правами Администратора со следующими включенными разрешениями:

Удаление сообщений (Delete messages) — чтобы бот мог стирать строчки добавления продуктов (п+ ...).

Закрепление сообщений (Pin messages) — чтобы бот мог вешать и обновлять список покупок в самом верху чата.

***📝 Примеры использования***

Добавление одного товара: п+ Молоко 2л

Добавление нескольких товаров через запятую: п+ Хлеб, Сыр 300гр, Бананы 1кг

Вычеркнуть/вернуть товар: Нажмите на кнопку с названием товара в списке.

Полная очистка: Нажмите кнопку 🗑 Очистить все купленные под списком.

Оповестить семью: Нажмите кнопку 🏃‍♂️ Я в магазине!.
